Cheat Sheet - GitHub

Step 1

Repository erstellen

Repository Code in VSC einfügen

    How to

  • ( git add . ) Fügt alle Änderungen im Arbeitsverzeichnis (neue, geänderte und gelöschte Dateien) dem Staging-Bereich hinzu. Schritte: Hinzufügen aller Änderungen im Arbeitsverzeichnis zum Staging-Bereich.
  • ( git commit -m "name" ) Erstellt einen neuen Commit mit den im Staging-Bereich befindlichen Änderungen und einer Commit-Nachricht. Schritte: Committet die im Staging-Bereich befindlichen Änderungen. Die -m Option ermöglicht es, eine Commit-Nachricht direkt in der Befehlszeile anzugeben, z.B., git commit -m "Name des Commits".
  • ( git push )Überträgt lokale Commits zum Remote-Repository auf GitHub (oder einem anderen Git-Host). Schritte: Übertragung der lokalen Commits auf das Remote-Repository. Andere Teammitglieder können dann diese Änderungen herunterladen, um auf dem neuesten Stand zu bleiben.

Step 2

branch anlegen, wechseln und löschen

    How to

  • ( git branch NAME ) Erstellt eine neue branch und gibt der branch einen Namen
  • ( git checkout NAME ) Wechselt zu dem branch wessen Namen du dahinter schreibst
  • ( git checkout -b NAME ) Erstellt eine branch mit Namen und wechselt sofort in die branch
  • ( git branch ) Zeigt alle vorhandenen branches an und "*" zeigt dir in welchem branch du dich befindest
  • ( git branch -d NAME ) Löscht den genannten branch

Step 3

Ablauf eines merges

Zusammenführen zweier branches

    How to

  • ( git merge ) Versucht den main branch mit deinem feature branch zusammenzufügen
  • ( git add . ) ( git commit -m "merge branch" ) (git push ) den zusammengefassten branch hochladen
  • fertige branches kann man löschen

Step 4

Commits (Version) zurück gehen

    How to

  • ( git log--oneline ) --oneline zeigt dir den zusammengefassten Verlauf von deinen commits an
  • ( git checkout 1234567 ) geht auf die mit der id genannte Version zurück
  • mit " Q " auf das normale Terminal schalten
  • ( git add . )( git commit -m "back" )(git push) dokumentiere dass du eine Version zurück bist
